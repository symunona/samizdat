## Markers on the scrub track (distinct colors + flag labels)
- **Resume** ("where I was") — green `colors.online`, flag label above the pole.
- **Jump-from** ("last jumped from") — amber `#f5b301`, flag label. Only the most-recent jump origin.
- **Annotation** — purple `#a78bfa` ticks, one per `media_ts_ms>0`, tappable → open that annotation.
Track = absoluteFill Pressable (seek) at the bottom; decorative bars/flags `pointerEvents="none"`
on top; annotation ticks are Pressables on top (capture their own taps). This also FIXES tap-seek
(item 5): decorations no longer steal `locationX`.

## Jump grace-window state machine (item 6)
`userSeek(toMs)` wraps every user seek (track tap, ±skip, transcript seg tap, annotation tap):
- captures `fromMs = positionRef.current`, seeks, and if `|toMs-fromMs| >= 500`:
  - `setJumpFromMs(fromMs)` (moves the amber flag to the newest origin)
  - `graceAnchorRef.current = toMs` (position we jumped TO)
- While `graceAnchorRef != null`, `savePosition()` skips persisting `video_pos_<docId>` and the
  resume marker doesn't advance. Grace clears once `positionMs - graceAnchor >= 10000` (≈10s of
  forward playback; a pause simply waits — interval only ticks while playing). A new jump resets the anchor.
- Mount auto-resume uses raw `seek()` (NOT a jump).

## Transcript ⊕ annotations (item 4, WebView)
- `buildTranscriptHtml` now emits `data-end-ms` per seg.
- `AnnData` gains `media_ts_ms?`. On init (transcript mode) + on `addMark`/`removeMark`, mark the seg
  whose `[start,end]` contains `media_ts_ms` (nearest by start otherwise): add `.has-ann`, a `data-ann-id`,
  and a ✏ badge (display-only, allowed per app/CLAUDE.md).
- Seg click: if `data-ann-id` → `sendMsg(tap_annotation, id)` (opens AnnotationPanel via existing host
  handler); else → existing `seek`. Text `mark` taps keep working.
- Run `just webview-build`. Parity lint: only the highlight-card pair is gated and its rendering is
  untouched → expected IN_SYNC; lint-parity is also a no-op pre-commit on main.
