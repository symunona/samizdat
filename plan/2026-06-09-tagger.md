# Tagger module

## Goals

1. **Tag any object** — a `Document` or an `Annotation` can have N tags.
2. **Tag selector** — modal that lists all tags; tap to toggle; "new tag" inline create.
3. **Tags screen** — in the left drawer; lists every tag with doc + annotation counts; tap → list of tagged objects.
4. **Navigation** — tapping a tagged object opens the document viewer; for annotations, scroll into view + apply bordered highlight style.

## Server: schema additions

```sql
CREATE TABLE IF NOT EXISTS tags (
  id         TEXT PRIMARY KEY,
  name       TEXT NOT NULL UNIQUE,
  color      TEXT NOT NULL DEFAULT 'default',
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  rev        INTEGER NOT NULL DEFAULT 0,
  deleted_at TEXT
);

CREATE TABLE IF NOT EXISTS document_tags (
  id          TEXT PRIMARY KEY,
  document_id TEXT NOT NULL REFERENCES documents(id),
  tag_id      TEXT NOT NULL REFERENCES tags(id),
  created_at  TEXT NOT NULL,
  rev         INTEGER NOT NULL DEFAULT 0,
  deleted_at  TEXT,
  UNIQUE(document_id, tag_id)
);

CREATE TABLE IF NOT EXISTS annotation_tags (
  id            TEXT PRIMARY KEY,
  annotation_id TEXT NOT NULL REFERENCES annotations(id),
  tag_id        TEXT NOT NULL REFERENCES tags(id),
  created_at    TEXT NOT NULL,
  rev           INTEGER NOT NULL DEFAULT 0,
  deleted_at    TEXT,
  UNIQUE(annotation_id, tag_id)
);
```

## Server: API routes

```
GET    /api/v1/tags                           list all tags (with doc_count, ann_count)
POST   /api/v1/tags                           create tag { name, color? }
DELETE /api/v1/tags/:id                       soft-delete tag

GET    /api/v1/tags/:id/documents             documents tagged with :id
GET    /api/v1/tags/:id/annotations           annotations tagged with :id

GET    /api/v1/documents/:id/tags             tags on a document
POST   /api/v1/documents/:id/tags             { tag_id } — add tag to document
DELETE /api/v1/documents/:id/tags/:tag_id     remove tag from document

GET    /api/v1/annotations/:id/tags           tags on an annotation
POST   /api/v1/annotations/:id/tags           { tag_id } — add tag to annotation
DELETE /api/v1/annotations/:id/tags/:tag_id   remove tag from annotation
```

## App: components

### TagSelectorModal
- Props: `visible`, `objectId`, `objectType: 'document'|'annotation'`, `onClose`
- Fetches all tags + object's current tags on open
- Shows list: each tag row = colored dot + name + checkmark if selected
- Tap to toggle (add/remove via API)
- "New tag" row at bottom: inline TextInput → POST /api/v1/tags
- Keyboard-avoiding, slide-up modal

### TagsScreen  (`app/(drawer)/tags.tsx`)
- Lists all tags sorted by total object count
- Each row: tag name + doc count chip + annotation count chip
- Tap → `TagDetailScreen` (can be same file, conditional render or separate route)

### TagDetailScreen (`app/(drawer)/tags/[id].tsx`)
- Header: tag name + total count
- Two sections: Documents, Annotations
- Document card: title + excerpt + annotation count
- Annotation card: excerpt of `exact` text + note snippet + parent doc title
- Tap document card → `/document/:id`
- Tap annotation card → `/document/:docId?highlight=:annId`

## App: deep-link highlight

`document/[id].tsx` reads `highlight` search param.  
When present:
- After load, inject `window.highlightAnnotation(annId)` into the iframe/WebView
- `highlightAnnotation(id)` in `html.go`:
  - finds `mark[data-ann-id=id]`
  - scrolls it into view
  - adds class `focused` → CSS: thicker outline + brighter background

CSS addition:
```css
mark.focused { outline: 2px solid currentColor; filter: brightness(1.4); }
```

## AnnotationPanel integration

`AnnotationPanel` receives optional `onTag` prop.  
The `# Tags` button calls `onTag(annotationId)` → parent opens `TagSelectorModal`.

## Drawer menu
