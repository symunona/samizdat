# CLAUDE.md — clipper/

Chrome extension (MV3). Capture client. POSTs to Samizdat server REST API.

## Shipped: "Save to Sam" (v2 — multi-instance + popup)
Popup dropdown + saved-state indicator + on-page toast. No capture/highlights/offline yet.
- `manifest.json` + `src/{background,content,popup,options}.js` + `src/{popup,options}.html` + `src/instances.js` (shared store). Icons generated by `gen-icons.js` (zero-dep PNG rasterizer, 3 states × 4 sizes) into `src/icons/` (gitignored).
- **Build:** `node build.js` (run via `just build-clipper`) → `dist/unpacked/` (load-unpacked) + `dist/sam-chrome.zip`. **No bundler yet** — zero npm deps, so build = gen icons + copy + zip. Introduce Vite only when capture (Defuddle/Turndown) lands and real bundling is needed.
- **Server:** zip at `GET /extension/sam-chrome.zip` (server `--extension-zip` flag). Version at `GET /api/v1/extension/version` → `{version}` read live from the served zip's `manifest.json` (no build/version coupling; 404 if zip unset). Health carries `app:"samizdat"` so the content script + popup recognise a Sam origin.
- **Multi-instance (v2):** storage is `instances: [{origin, hostname, token}]` — keyed by **origin**, labelled by **hostname** (host:port, so `localhost:8765` and prod stay distinct). `src/instances.js` (IIFE → `globalThis.SamInstances`) owns the store + v1→v2 migration (folds legacy `serverBase`/`deviceToken` into one instance). Loaded via `importScripts` in the SW, `<script>` in popup/options, and as a content_script before `content.js`. **IIFE-wrapped** because classic scripts share one global lexical scope — a bare top-level `const` would collide with the same name destructured by consumers.
- **Popup (`action.default_popup`):** dropdown, no more bare `onClicked`. One row per instance — single instance → "Save as document to Samizdat"; multiple → "Save to `<hostname>`". Saved URLs show ✓ "Open in …". On a connected Sam tab (host page) → extra items: Open dashboard / Manage instances / Disconnect. On an unconnected Sam origin (live health probe) → "Connect …". Newer server bundle → "Download new version (vX)". Saving runs in the **background** (outlives the popup) → polls by-url → `chrome.tabs.sendMessage` a shadow-DOM **toast** ("URL added to stream") into the page.
- **Badge:** when >1 instance connected, the toolbar shows the **instance count**; transient save badges (…/✓/!) override it then restore.
- **Auto-pair:** the web Settings page mints a device token via `POST /api/v1/devices/extension-token` (**Bearer-authed by the web app's own device token — no cookie session**), then `window.postMessage`s it to `content.js`, which **appends** the origin to `instances` (many instances may pair over time). Marker `data-sam-ext = "<version>:<paired|unpaired>"` (app's settings.tsx checks `endsWith(':paired')` — keep that suffix exact).
- **Caveats (acceptable for single-user dogfood, fix later):**
  - A malicious site faking `/api/v1/health` can surface a "Connect" row, but connecting still requires the real Settings page to mint a token — no token leaks. Origin-trust hardening deferred.
  - dedup uses `GET /documents/by-url` on the **raw tab URL** (exact `canonical_url` match) — occasional false "not saved" when the server canonicalises differently.

## MV3 hard constraints
- **No remote code execution** — all logic must ship in the extension bundle
- **No eval, no dynamic import from remote URLs**
- Background script = service worker (`background.js`), not a persistent page
- Site adapters are JSON config data shipped in the bundle, not remote JS

## Extraction pipeline (client-side only)
1. Defuddle — main content extraction (fork Obsidian Web Clipper patterns)
2. Turndown — HTML → Markdown conversion
3. Result POSTed to server as markdown; server never receives raw HTML

## Offline queue
IndexedDB queue for failed POSTs. Drain on next successful connection.
Never lose a clip due to network failure.

## API
- `POST /api/v1/documents` — `{url, markdown, title, captured_at}`
- Bearer token stored in `chrome.storage.local` — **never** `chrome.storage.sync` (token is a secret)

## Conventions
- No frameworks in content scripts (plain DOM APIs)
- Popup and options page may use a lightweight framework if needed (decide before first UI component)
- Vite for bundling
- Adapters = JSON files in `src/adapters/` — one per site domain pattern
